Paper Validation Battery

Four targeted experiments to make the protocol study statistically defensible for publication. gpt-4o-mini throughout.
Global Configuration
Suite A Required Statistical backbone — n=20 replication resolves sample size issue  ·  Suite B Required NATURAL-TERSE control — isolates verbosity vs format effect  ·  Suite C Required Discovery task — tests if NATURAL wins when field list is unknown  ·  Suite D Required Cross-task generalization — GO/NO-GO clinical cases, NUMERIC v2 vs baselines
A
Scaled Head-to-Head (n=20)
NATURAL · STRUCTURED · NUMERIC v2
Statistical backbone. Exact replication of Experiment 5 at n=20 per condition. Addresses the sample size problem — n=5 cannot support significance tests. At n=20 we can run t-tests and compute 95% CIs on token counts and accuracy. Same VC task, same dataset, same protocol designs.
B
NATURAL-TERSE Control
NATURAL · NATURAL-TERSE · STRUCTURED · NUMERIC v2
Verbosity confound resolution. Previous experiments showed NATURAL agents spend ~3× more tokens than STRUCTURED. But is that because of the format, or because NL agents choose to write paragraphs? NATURAL-TERSE uses identical prompts but constrains the tasker to one sentence per field — no explanatory prose. If NATURAL-TERSE approaches STRUCTURED efficiency, the cost is verbosity choice, not format. If it doesn't, the cost is intrinsic to the format.
C
Open-Ended Task (No Field List)
NATURAL · STRUCTURED · NUMERIC v2
NATURAL advantage test. In all prior experiments, solvers knew the available field names in advance. This experiment removes the field list from all solvers and requires discovery. NATURAL can ask free-form questions. STRUCTURED can query guessed fields or broad categories. NUMERIC v2 can query guessed field names or equivalent category tokens, but has no wildcard dump-all command. Tests whether NATURAL's broad schema is an advantage when the information space is genuinely unknown.
D
Cross-Task Generalization — Clinical Trial
NATURAL · STRUCTURED · NUMERIC v2
All five prior experiments used the same VC investment task. Suite D rotates three clinical-trial scenarios: clear GO, clear NO-GO, and mixed-risk NO-GO. 8 fields instead of 12. Different domain knowledge. Different correct answer signals (efficacy, safety, completion, precedent). Terminal verbs: GO or NO-GO. Tests whether NUMERIC v2's efficiency and accuracy hold across task domains and both decision labels.
